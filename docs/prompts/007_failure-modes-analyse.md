# Prompt 007 – Failure Modes Analyse

**Phase:** Paket 5 – API-Stabilität

## Prompt

Kontext:
Import- und Read-Pipeline muss robust werden.

Analysiere:
1. Duplicate Inserts
2. Parser liefert leeren Text
3. PDF ohne Text / Scan
4. sehr große Dokumente
5. Chunk Explosion
6. DB Timeout / Locking

Für jeden Punkt:
- Ursache
- aktuelles Verhalten
- Risiko
- Fix / Mitigation

Output:
- Failure Mode Matrix
